How to log all client ip addresses in Spring

🗓 2019-03-16
⏱ 예상 소요시간 - [7 분]
aop, aspect, logback, logging, slf4j

Intro

스프링에서 요청한 클라이언트의 IP를 로그로 기록하는 방법에 대해 정리한다.

여기서는 다음을 알고 있다고 가정한다.

  • 인터셉터
  • AOP
  • slf4j, logback 및 설정 방법
  • MDC(Mapped Diagnostic Context)

1. Approach

일반적으로 logback을 사용해서 로그를 기록하면 클라이언트 IP는 보이지 않는다. 따라서 다음을 사용한다.

  • MDC
  • Interceptor(or Filter)
  • Aspect

MDC의 문서를 보면 put 메소드로 값을 넣어 로그에서 출력할 수 있다.
매 요청에 대해 기록하고 싶다면 Interceptor를 사용하고 필요한 곳에만 기록하고 싶다면 Aspect를 사용한다.

3. 인터셉터

인터셉터의 preHandle를 구현해서 등록해두면 컨트롤러를 처리하기 전에 호출되므로 여기서 IP를 얻어서 기록하면 된다.

nginx와 같은 로드 밸런서 뒤에 있는 인스턴스면 HTTP 헤더의 X-FORWARDED-FOR에 기록되기 때문에 이런 작업이 필요하다.
하지만 인터셉터를 사용하면 매 요청마다 IP가 기록된다. 필요하면 그대로 써도 좋지만 선택적으로 기록하고 싶은 경우도 있을 것이다.
이런 경우에 Aspect를 사용하면 적절하다.

4. Aspect

선택적으로 사용하기 위해 어노테이션을 사용할 것이다. 우선 어노테이션부터 작성해보자.

적용 대상이 클래스(타입)와 메소드이다. 따라서 부분적으로 원하는 곳에 적용을 할 수 있다. 이제 Aspect를 작성해보자.

어노테이션의 적용 대상이 타입과 메소드였기 때문에 포인트컷 지정자(pointcut designator)에서 within과 annotation을 사용했다.
within은 특정 타입에 대하여 조인포인트를 제한할 수 있다. annotation은 말 그대로 어노테이션이다.

이제 그저 사용하기만 하면 된다.
